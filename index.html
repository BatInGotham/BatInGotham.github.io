嵌入式CPU——MCU

MCU：一定含有外设，且至少包含定时器和GPIO，一般还具有ADC、UART和PWM等。

CPU：只有指令译码部分，不含有任何外设，如x86

SoC：带有存储器及某个特定外设的MCU（如esp8266）

嵌入式CPU分类

  • 8 bit vs 32 bit
  • CISC vs RISC
  • not ARM vs ARM
  • without MMU vs with MMU

下载方式

  • 片上存储器
  • 外接存储器

功能

  • GPIO：数量、驱动能力
  • UART：数量
  • AD：分辨率、速度、数量
  • 定时器：数量、长度
  • SPI：速度
  • I2C：速度
  • 其他通信：CAN、USB

封装

BGA：体积小、价格低、加工成本高

QFP：加工成本低、适合手工焊接

DIP：体积大、价格高

2019/03/12 Operating System

2.1 进程与线程 知识概要

进程的概念

引入进程的概念是为了更好地描述和控制程序的并发执行，实现操作系统的并发性和共享性。提高资源利用率。

进程映像（进程实体）

由程序段、相关数据段、PCB三部分构成了进程映像（进程实体）。创建进程就是创建进程映像中的PCB，撤销进程就是撤销进程的PCB。进程映像是静态的，进程是动态的。

Q1：进程、进程实体、程序之间的关系？

PCB是进程存在的唯一标识。

进程的定义

进程是进程实体的运行过程，是系统进行资源分配和调度的一个独立单位。

进程的特征

  • 动态性：进程是程序的一次执行，与程序（进程映像）的根本区别在于进程是动态的。
  • 并发性：多个进程实体同时存在在内存中，能在一段时间内同时运行。
  • 独立性：进程实体是一个能独立运行、独立获得资源、独立接受调度的基本单位。
  • 异步性：进程相互制约，进程具有执行的间断性，即进程按各自独立的、可预知的速度向前推进。
  • 结构性：每个进程都配置一个PCB对其进行描述。

进程的转换

  1. 运行态 -> 就绪态：运行态中的进程，时间片用完/被抢占后，会转换到就绪态。
  2. 就绪态 -> 运行态：处于就绪态中的程序得到调度，获得处理器资源后进程变为运行态。
  3. 运行态 -> 阻塞态：进程请求某一资源的使用，或等待某一事件的发生时，会从运行态转变成阻塞态。系统调用就是这一过程。
  4. 阻塞态 -> 就绪态：进程等待的事件到来时，中断处理程序把相应的进程转到就绪态。

进程的创建

父进程创建子进程，子进程可以继承父进程的所有资源，子进程被撤销时应把从父进程处获得的资源归还。撤销父进程时必须撤销其所有子进程。

原语描述：

  1. 分配PID，申请一个空白PCB。若PCB申请失败，则创建失败。
  2. 为进程分配资源，分配内存空间。如果内存不足，则进入阻塞态（请求必要的系统资源）。
  3. 初始化PCB。
  4. 若就绪队列可以接纳新进程，则插入就绪队列，等待调度。

进程的终止

引起进程终止的原因有：任务完成、异常事件、外界干预。

  1. 根据标识符找到PCB，读取进程的状态。
  2. 若在执行状态，则终止该进程的执行，将处理机资源分配给其他进程。
  3. 若该进程还有子进程，终止所有子进程。
  4. 将该进程所拥有的全部资源归还给其父进程或操作系统。
  5. 将PCB从所在队列（链表）中删除。

进程的阻塞和唤醒

阻塞原语（Block）的描述如下：

  1. 找到阻塞进程的PCB
  2. 若该进程在运行态，则保护其现场，将其状态转为阻塞态并停止运行。
  3. 把PCB插入相应事件的等待队列。

唤醒（Wakeup）原语：

  1. 把该事件的等待队列中找到相应进程的PCB。
  2. 将其从等待队列中移出，并置其状态为就绪态。
  3. 把该PCB插入就绪队列，等待调度程序调度。

进程切换

  1. 保存处理机上下文（程序计数器、寄存器等）。
  2. 更新PCB。
  3. 把进程的PCB移入相应队列，如就绪、在某事件阻塞等队列。
  4. 选择另一个进程执行，并更新其PCB。
  5. 更新内存管理的数据结构。
  6. 恢复处理机上下文。

进程的组织

  • PCB：包含进程描述信息、进程控制和管理信息、资源分配清单、处理机相关信息；PCB常用的组织方式有链接方式和索引方式。
  • 程序段：程序可被多个进程共享
  • 数据段

进程的通信

  • 共享存储（Share Memory）
  • 消息传递（Message）
  • 管道通信（Pipe）：注意管道机制是半双工通信，即某一时刻只能单向传输

线程

线程是“轻量级进程”，引入进程的目的是为了更好地使多道程序并发执行，提高资源利用率和系统吞吐量，增加并发程度；引入线程的目的是减小程序在并发执行时所付出的时空开销，提高操作系统的并发性能。

线程由线程ID、程序计数器、寄存器集合、堆栈组成。线程是进程中的一个实体，是被系统独立调度和分派的基本单位。

引入线程后，进程只作为除了CPU以外的系统资源的分配单元，线程作为处理机的分配单元。

Chapter 4 Top-Down Parsing

Recursive-Descent Parsing

The idea of Recursive-Descent Parsing

  • The grammar rule for a non-terminal A as a definition for a procedure to recognize an A
  • The right-hand side of the grammar for A specifies the structure of the code for this procedure

The Basic Method of Recursive Descent

  • Writing recursive-decent procedure for the remaining rules in the expression grammar is not as easy for factor.
  • It requires the use of EBNF.

Repetition and Choice: Using EBNF

Example: expr -> expr addop term|term

this would lead to an immediate infinite recursive loop

the solution is use the EBNF rule: expr -> term {addop term}

The recursive-descent method is quite powerful and adequate to
construct a complete parse. But we need more formal methods
to deal with complex situation.

Further Decision Problems

The recursive-descent method is quite powerful and adequate to construct a complete parse.

But we need more formal methods to deal with complex situation.

  1. It may be difficult to convert a grammar in BNF into EBNF form.
  2. It is difficult to decide when to use the choice A -> a and the choice A -> b; if both a and b begin with non-terminals (First Sets)
  3. It may be necessary to know what token legally coming from the non-terminal A, A->e (Follow Sets)
  4. It requires computing the First and Follow sets in order to detect the errors as early as possible

LL(1) Parsing

Main idea: LL(1) Parsing uses an explicit stack rather than recursive calls to perform a parse.

Want to decide which production to apply based on next symbol

The two actions:

  1. Generate: replace a non-terminal A at the top of the stack by a string a(in reverse) using a grammar rule A -> a
  2. Match: match a token on top of the stack with the next input token

Top-Down Parsing == Generate & Match Parsing

Constructing a parse tree:
Adding node construction actions as each non-terminal or terminal is push onto the stack.

LL(1) Parsing table

S -> (S) S | e

M[N, T] ( ) $
S S->(S)S S->e S->e

The general LL(1) Parsing table definition: The table is a two-dimensional array indexed by non-terminals
and terminals The table contains production choices to use at the appropriate parsing step, which called M[N,T].

  • N is the set of non-terminals of the grammar
  • T is the set of terminals or tokens (including $)
  • Any entrances remaining empty represent potential errors

Ambiguity（歧义）

calculate priority lead to ambiguity.

离root越远，运算优先级越高

判断歧义文法：

设计一个sentence，如果可以构造两个结构不同的syntax tree，则为歧义文法

An ambiguous grammar: a grammar that generates a string with two distinct parse trees is called an ambiguous grammar.

Such a grammar represents a serious problem for a parser. It does not specify precisely the syntactic structure of a program.

Two basic methods

  1. A rule: that specifies in each ambiguous case which of the parse trees (or syntax trees) is the correct one. Such a rule is called a disambiguating rule.
    • The advantage: it corrects the ambiguity without
      changing (and possibly complicating) the grammar.
    • The disadvantage: the syntactic structure of the
      language is no longer given by the grammar alone.
  2. Change the grammar into a form that forces the construction of the correct parse tree.
    • Adv. and Disadv. are converted.

Remove the ambiguity

  • State a disambiguating rule that established the relative precedences of the three operations represented.
  • The associativity of each of the operations of addition, subtraction, and multiplication

Precedence and associativity

both may cause ambiguity.

precedence: 1+2*3

ambiguity: 1-2-3

The precedence cascades cause the parse trees to become much more complex. The syntax trees, however, are not affected.

The dangling else problem

An else-part should always be associated with the nearest if-statement that does not yet have an associated else-part.

In C, we may use brackets.

也可以改造文法

To design the syntax that the dangling else problem does not appear.

  1. Require the presence of the else-part, and this method has been used in LISP and other functional languages (where a value must also be returned).
  2. Use a bracketing keyword for the if-statement languages that use this solution include Algol68 and Ada

Inessential ambiguity

Sometimes a grammar may be ambiguous and yet always produce unique abstract syntax trees.

Inessential ambiguity: the associated semantics do not depend on what disambiguating rule is used.

Extended Notations: EBNF and syntax diagrams

BNF notation is sometimes extended to include special notations for repetitive and optional constructs.

A -> b{a}

A -> {a}b

Optional constructs in EBNF are indicated by surrounding them with square brackets […].

Formal properties of context-free language

Definition : A context-free grammar consists of the following:

  1. A set T of terminals.
  2. A set N of nonterminals (disjoint from T).
  3. A set P of productions, or grammar rules, of the form A -> a, where A is an element of N and a is an element of (T union N)* (a possibly empty sequence of terminals and nonterminals).
  4. A start symbol S from the set N.

Let G be a grammar as defined above, G = (T, N, P, S).

  • A derivation step over G is of the form a A y => a b y ,
    where a and y are elements of (T union N)* and A -> y is in P.
  • The set of symbols: The union T
  • N of the sets of terminals and nonterminals
  • A sentential form: a string a in (T union N)*.
  • a => *b if and only if there is a sequence of 0 or more derivation steps (n>=0)
  • A derivation over the grammar G is of the form S => w, where w belongs to T

１. 计算机网络分为局域网、广域网、城域网，其划分的主要依据是（ ）

A. 网络的作用范围

B. 网络的拓扑结构

C. 网络的通信方式

D. 网络的传输性质

答案选A

划分局域网、广域网、城域网的依据是网络的作用范围，三者的关键区别在于网络协议的不同

2. 局域网和广域网的差距不仅在于它们所覆盖的范围不同，还主要在于（ ）

B. 所使用的协议不同

C. 所能支持的通信量不同

D. 所提供的服务不同

答案选B

主要区别还是网络协议的不同

3. 下列说法正确的是（ ）

A. 在较小的范围内布置的一定是局域网，在较大范围内布置的一定是广域网

B. 城域网是连接广域网而覆盖园区的网络

C. 城域网是为淘汰局域网和广域网而提出的新技术

D. 局域网是基于广播技术发展起来的网络，广域网是基于交换技术发展起来的网络

答案选D

A选项，范围不是决定因素，协议是；B选项，城域网连接的是局域网；C选项不存在，城域网是符合特定场合而出现的

4. 现在大量的计算机是通过诸如以太网这样的局域网连入广域网的，而局域网与广域网的互联是通过（ ）实现的

A. 路由器

B. 资源子网

C. 桥接器

D. 中继器

WAN和LAN的互联是通过路由器；桥接器和中继器是指局域网的物理层和链路层的联网设备

5. 下列关于广播式网络的说法，错误的是（ ）

A. 共享广播信道

6. 下列（ ）是分组交换网络的缺点

C. 传播时延大

有连接、可靠的数据报服务是由传输层的TCP实现的

14. 假设OSI参考模型的应用层欲发送400 Byte的数据（无拆分），除物理层和应用层外，其他各层在封装PDU时均引入20 Byte的额外开销，则应用层的数据传输效率约为（ ）

答案是80%

OSI模型共7层，除去物理层和应用层，还有5层，共5*20=100 Byte的额外开销，400/500 = 80%
