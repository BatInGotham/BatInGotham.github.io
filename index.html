PAT-A1112 Solution

PAT-A1112: Stucked Keyboard

Problem Description

On a broken keyboard, some of the keys are always stucked. So when you type some sentences, the characters corresponding to those keys will appear repeatedly on screen for k times.

Now given a resulting string on screen, you are supposed to list all the possible stucked keys, and the original string.

Notice that there might be some characters that are typed repeatedly. The stucked key will always repeat output for a fixed k times whenever it is pressed. For example, when k=3, from the string thiiis iiisss a teeeeeest we know that the keys i and e might be stucked, but s is not even though it appears repeatedly sometimes. The original string could be this isss a teest.

Input Specification

Each input file contains one test case. For each case, the 1st line gives a positive integer k (1<k≤100) which is the output repeating times of a stucked key. The 2nd line contains the resulting string on screen, which consists of no more than 1000 characters from {a-z}, {0-9} and _. It is guaranteed that the string is non-empty.

Output Specification

For each test case, print in one line the possible stucked keys, in the order of being detected. Make sure that each key is printed once only. Then in the next line print the original string. It is guaranteed that there is at least one stucked key.

题意

给定一个k，某些坏掉的按键按一下会输出k次，给定输出序列，按顺序打印最多情况下可能的坏键，并给出对应的输入序列。

本题比较容易出问题的点有如下几种：

  1. 序列eee_e，e一定是好键
  2. 输出按键顺序是根据输入中的位置

较优解法

思想：状态机

开一个int数组，对应字符的ASCII码，初始化值为0。

  • 0：start state，表示未知好坏
  • 1：accept state，表示已知是好的
  • 2：normal state，表示可能是坏的

在第二次遍历时额外引入状态3来表示有无被遍历过

从头到尾遍历（第一次）输入序列，对每个一个输出字符，如果从该字符开始有连续k个相同输出，且字符状态不为1，则置该字符状态为2，迭代变量向后跳k个；如果没有连续k各相同输出，则置字符状态为1，且状态自此以后不变。

第二次遍历，对所有的字符进行状态查询，如果为2，则输出。

第三次遍历，对所有字符进行状态查询，如果为1，则输出一次并跳进k个单位，如果不为1，则输出一次并跳进1个单位。

Chapter 4 (cont.)

First and Follow Sets

First Sets

Definition:

Let X be a grammar symbol (a terminal or non-terminal) or epsilon. Then First(X) is a set of terminals of epsilon, which is defined as follows:

  1. If X is a terminal or epsilon, then First(X) = {X};
  2. If X is a non-terminal, then for each production choice X -> X1 X2 … Xn, First(X) contains First(X1) - {epsilon}.

Let α = X1 X2 … Xn be a string of terminals and non-terminals,

First(α) is defined as follows:

  1. First(α) contains First(X1) - {epsilon};
  2. For each i=2…n, if for all k=1,…,i-1, First(X_k) contains epsilon, then First(α) contains First(X_k) - {epsilon};
  3. If all the set First(X1)…First(Xn) contain epsilon, the First(α) contains epsilon.

A non-terminal A is nullable if there exists a derivation A => *epsilon

Theorem: A non-terminal A is nullable if and only if First(A) contains epsilon.

Follow Sets

Definition:

Given a non-terminal A, the set Follow(A) is defined as follows.

  1. If A is the start symbol, the $ is in the Follow(A).
  2. If there is a production B -> αAγ, then First(γ) - {epsilon} is in Follow(A).
  3. If there is a production B -> αAγ, such that epsilon in First(γ), then Follow(A) contains Follow(B).

Notes:

  1. The symbol $ is used to mark the end of the input
  2. The empty “pseudo-token” epsilon is never an element of a follow set
  3. Follow sets are defined only for non-terminal
  • Follow sets work “on the right” in production α
  • First sets work “on the left” in the production.
  • Given a grammar rule A -> αB, Follow(B) will contain Follow(A),the opposite of the situation for first sets, if A -> B α, First(A) contains First(B),except possibly for ε.

Use First and Follow Sets Constructing LL(1) Parsing Tables

Error Recovery

Different levels of response to errors.

Given a meaningful error message

Determine as closely as possible the location where that error has occurred

Some form of error correction (error repair)

The parser attempts to infer a correct program from the incorrect one given

Most of the techniques for error recovery are ad hoc, and general principles are hard to come by.

Some important considerations that apply are the following

  1. To determine that an error has occurred as soon as possible.
  2. After an error has occurred, pick a likely place to resume the parse try to parse as much of the code as possible
  3. To avoid the error cascade problem
  4. To avoid infinite loops on error

The important decisions to be made in this error recovery method

  1. What tokens to add to the synchronizing set at each point in the parse?
    Follow sets are important candidates
  2. First sets:
    • Prevent the error handler from skipping important tokens that begin major new constructs
    • Detect errors early in the parse.

Regular Expression 正则表达式

正则表达式表示字符串的格式。正则表达式r完全由它所匹配的串集来定义。这个集合称为由正则表达式生成的语言，写作L（r）。该语言依赖于适用的字符集，它一般是ASCII字符的集合或它的某个子集，此处集合的元素被称为符号（symbol），这个正规符号的集合称为字母表（alphabet），用希腊字母Σ表示。

正规表达式r还包括字母表中的字符，但这些字符具有不同的含义：在正则表达式中，所有的符号指的都是模式。

正则表达式r中，还可包含有特殊含义的字符，这样的字符称为元字符（meta-character）或元符号（meta-symbol）。它们并不是字母表中的正规字符，否则就难以区分其作为元字符或字母表中的字符了。

正则表达式的定义

1. 基本正则表达式

它们是字母表中的单个字符且自身匹配。有时需要使用空串，空串就是不包含任何字符的串，用epsilon表示。

2. 正则表达式运算

正则表达式中有三种基本运算：

  • 从各选择对象中选择，用元字符|表示
  • 连结，由并置表示（就是连续排列）
  • 重复或闭包，由元字符*定义

2019/03/12 Operating System

2.1 进程与线程 知识概要

进程的概念

引入进程的概念是为了更好地描述和控制程序的并发执行，实现操作系统的并发性和共享性。提高资源利用率。

进程映像（进程实体）

由程序段、相关数据段、PCB三部分构成了进程映像（进程实体）。创建进程就是创建进程映像中的PCB，撤销进程就是撤销进程的PCB。进程映像是静态的，进程是动态的。

Q1：进程、进程实体、程序之间的关系？

PCB是进程存在的唯一标识。

进程的定义

进程是进程实体的运行过程，是系统进行资源分配和调度的一个独立单位。

进程的特征

  • 动态性：进程是程序的一次执行，与程序（进程映像）的根本区别在于进程是动态的。
  • 并发性：多个进程实体同时存在在内存中，能在一段时间内同时运行。
  • 独立性：进程实体是一个能独立运行、独立获得资源、独立接受调度的基本单位。
  • 异步性：进程相互制约，进程具有执行的间断性，即进程按各自独立的、可预知的速度向前推进。
  • 结构性：每个进程都配置一个PCB对其进行描述。

进程的转换

  1. 运行态 -> 就绪态：运行态中的进程，时间片用完/被抢占后，会转换到就绪态。
  2. 就绪态 -> 运行态：处于就绪态中的程序得到调度，获得处理器资源后进程变为运行态。
  3. 运行态 -> 阻塞态：进程请求某一资源的使用，或等待某一事件的发生时，会从运行态转变成阻塞态。系统调用就是这一过程。
  4. 阻塞态 -> 就绪态：进程等待的事件到来时，中断处理程序把相应的进程转到就绪态。

进程的创建

父进程创建子进程，子进程可以继承父进程的所有资源，子进程被撤销时应把从父进程处获得的资源归还。撤销父进程时必须撤销其所有子进程。

原语描述：

  1. 分配PID，申请一个空白PCB。若PCB申请失败，则创建失败。
  2. 为进程分配资源，分配内存空间。如果内存不足，则进入阻塞态（请求必要的系统资源）。
  3. 初始化PCB。
  4. 若就绪队列可以接纳新进程，则插入就绪队列，等待调度。

进程的终止

引起进程终止的原因有：任务完成、异常事件、外界干预。

  1. 根据标识符找到PCB，读取进程的状态。
  2. 若在执行状态，则终止该进程的执行，将处理机资源分配给其他进程。
  3. 若该进程还有子进程，终止所有子进程。
  4. 将该进程所拥有的全部资源归还给其父进程或操作系统。
  5. 将PCB从所在队列（链表）中删除。

进程的阻塞和唤醒

阻塞原语（Block）的描述如下：

  1. 找到阻塞进程的PCB
  2. 若该进程在运行态，则保护其现场，将其状态转为阻塞态并停止运行。
  3. 把PCB插入相应事件的等待队列。

唤醒（Wakeup）原语：

  1. 把该事件的等待队列中找到相应进程的PCB。
  2. 将其从等待队列中移出，并置其状态为就绪态。
  3. 把该PCB插入就绪队列，等待调度程序调度。

进程切换

  1. 保存处理机上下文（程序计数器、寄存器等）。
  2. 更新PCB。
  3. 把进程的PCB移入相应队列，如就绪、在某事件阻塞等队列。
  4. 选择另一个进程执行，并更新其PCB。
  5. 更新内存管理的数据结构。
  6. 恢复处理机上下文。

进程的组织

  • PCB：包含进程描述信息、进程控制和管理信息、资源分配清单、处理机相关信息；PCB常用的组织方式有链接方式和索引方式。
  • 程序段：程序可被多个进程共享
  • 数据段

进程的通信

  • 共享存储（Share Memory）
  • 消息传递（Message）
  • 管道通信（Pipe）：注意管道机制是半双工通信，即某一时刻只能单向传输

线程

线程是“轻量级进程”，引入进程的目的是为了更好地使多道程序并发执行，提高资源利用率和系统吞吐量，增加并发程度；引入线程的目的是减小程序在并发执行时所付出的时空开销，提高操作系统的并发性能。

线程由线程ID、程序计数器、寄存器集合、堆栈组成。线程是进程中的一个实体，是被系统独立调度和分派的基本单位。

引入线程后，进程只作为除了CPU以外的系统资源的分配单元，线程作为处理机的分配单元。
