Ambiguity（歧义）

calculate priority lead to ambiguity.

离root越远，运算优先级越高

判断歧义文法：

设计一个sentence，如果可以构造两个结构不同的syntax tree，则为歧义文法

An ambiguous grammar: a grammar that generates a string with two distinct parse trees is called an ambiguous grammar.

Such a grammar represents a serious problem for a parser. It does not specify precisely the syntactic structure of a program.

Two basic methods

  1. A rule: that specifies in each ambiguous case which of the parse trees (or syntax trees) is the correct one. Such a rule is called a disambiguating rule.
    • The advantage: it corrects the ambiguity without
      changing (and possibly complicating) the grammar.
    • The disadvantage: the syntactic structure of the
      language is no longer given by the grammar alone.
  2. Change the grammar into a form that forces the construction of the correct parse tree.
    • Adv. and Disadv. are converted.

Remove the ambiguity

  • State a disambiguating rule that established the relative precedences of the three operations represented.
  • The associativity of each of the operations of addition, subtraction, and multiplication

Precedence and associativity

both may cause ambiguity.

precedence: 1+2*3

ambiguity: 1-2-3

The precedence cascades cause the parse trees to become much more complex. The syntax trees, however, are not affected.

The dangling else problem

An else-part should always be associated with the nearest if-statement that does not yet have an associated else-part.

In C, we may use brackets.

也可以改造文法

To design the syntax that the dangling else problem does not appear.

  1. Require the presence of the else-part, and this method has been used in LISP and other functional languages (where a value must also be returned).
  2. Use a bracketing keyword for the if-statement languages that use this solution include Algol68 and Ada

Inessential ambiguity

Sometimes a grammar may be ambiguous and yet always produce unique abstract syntax trees.

Inessential ambiguity: the associated semantics do not depend on what disambiguating rule is used.

Extended Notations: EBNF and syntax diagrams

BNF notation is sometimes extended to include special notations for repetitive and optional constructs.

A -> b{a}

A -> {a}b

Optional constructs in EBNF are indicated by surrounding them with square brackets […].

Formal properties of context-free language

Definition : A context-free grammar consists of the following:

  1. A set T of terminals.
  2. A set N of nonterminals (disjoint from T).
  3. A set P of productions, or grammar rules, of the form A -> a, where A is an element of N and a is an element of (T union N)* (a possibly empty sequence of terminals and nonterminals).
  4. A start symbol S from the set N.

Let G be a grammar as defined above, G = (T, N, P, S).

  • A derivation step over G is of the form a A y => a b y ,
    where a and y are elements of (T union N)* and A -> y is in P.
  • The set of symbols: The union T
  • N of the sets of terminals and nonterminals
  • A sentential form: a string a in (T union N)*.
  • a => *b if and only if there is a sequence of 0 or more derivation steps (n>=0)
  • A derivation over the grammar G is of the form S => w, where w belongs to T
