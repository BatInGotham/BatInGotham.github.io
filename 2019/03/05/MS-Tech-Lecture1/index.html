Module 2: 数据中心环境

SATA

逻辑数据块寻址

机械磁盘：柱面×磁道×扇区=逻辑块个数

磁盘驱动器性能

  • 电子机械设备

    影响磁盘总体性能

  • 磁盘服务时间

    I/O请求所需要的时间=寻道时间+旋转延迟+数据传输速度

    平均旋转延迟=旋转一周所用时间的一半

    对于X’’rpm，驱动器延迟 = (1/2) / (X/60)

数据传输速度

基于应用程序要求和磁盘驱动器性能的存储设计

  • 满足应用容量要求Dc

  • 满足应用性能要求Dp

  • 根据磁盘服务时间由磁盘提供服务的IOPS = 1/Ts

    其中Ts为I/O时间

应用程序所需要的磁盘 = max(Dc, Dp)

闪存

  • 没有机械运动，驱动器吞吐量达到最高
  • I/O延迟相对比较低
  • 高能效

主机对存储的访问方式

  1. 直接连接
  2. 网络连接（数据块级请求/文件级请求）

Module 3: 数据保护——RAID

Redundant Arrays of Independent Disks

软件实现&硬件实现

一般结构：主机-RAID控制器-磁盘逻辑阵列

RAID技术

  • 分条 Striping

    将多个磁盘的同一层级连成一个条Stripe

    提升性能，增加容量

  • 镜像 Mirror

    I/O效率提升，有冗余，浪费空间

  • 奇偶校验

    增加一个奇偶校验盘，逆向算出坏盘，节约空间（相比Mirror）

    奇偶校验一般是基于分条的

RAID分类

  • RAID 0 - 无容错能力的分条集

    e.g. 任意数据切成五块，同时写入五个盘；读数据也是同时读（并行读写）

    好处：I/O最快

    坏处：坏一个盘会立刻导致数据都失效

  • RAID 1 - 磁盘镜像

    e.g. 复制一遍，支持同时读写

    好处：安全，有一套备份

    坏处：浪费一半的空间

  • RAID 0+1/1+0 - 嵌套RAID

    0+1 条带集组成的镜像集

    e.g. 四个盘分为两组，两组一模一样，都进行条带化，AB数据的存储变为(AB)(AB)

    结合了0和1的优点，但是仍然浪费了一半的空间

    1+0 镜像集组成的条带集

    e.g. 四个盘分为两组，AB数据的存储变为(AA)(BB)

    相比0+1更优秀，不会因为一个盘失效而崩溃。

  • RAID 3 - 具有并行访问和专用奇偶校验的分条集

    顺序

  • RAID 4 - 具有独立磁盘访问和专用奇偶校验的分条集

    相比RAID 3可以独立磁盘访问，加快I/O

  • RAID 5 - 具有独立磁盘访问和分布式奇偶校验的分条集

    奇偶校验分散到各个盘中，降低奇偶校验盘失效的风险

  • RAID 6 - 具有独立磁盘访问和双分布式奇偶校验的分条集

适用范围（一般）

  • RAID 1+0 小型、随机、写入密集型
  • RAID 3 大型、顺序读入
  • RAID 5/6 小型、随机工作负载
